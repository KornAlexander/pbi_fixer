Remember my Power BI Fixer v1? The PowerShell script that could batch-fix entire workspaces — replacing pie charts, applying themes, adjusting page sizes? That was version 1. A proof of concept. A glimpse into what's possible.

Version 2 is a completely different animal.

The Power BI Fixer v2 is a full-featured, interactive development environment for Power BI reports and semantic models — running entirely inside a Microsoft Fabric Notebook. No installs. No downloads. No license costs. No desktop tools required. Just one line of Python:

from sempy_labs import pbi_fixer
pbi_fixer()

That's it. A complete UI appears directly in your notebook cell output — with 12 tabs, 45+ automated fixers, a model explorer, a report explorer, a memory analyzer, best practice analyzers, a translation editor, a prototyping tool, and much more.

Why Does This Exist?

Let me be honest. I built this because I was frustrated. Every single Power BI engagement I ran as a Solution Engineer followed the same pattern: open the customer's report, find the same 20 issues, explain the same best practices, manually fix the same problems. Over and over.

The existing tools in the ecosystem are excellent — but they require desktop installations, paid licenses, or both. They're designed for developers building reports from scratch. What I needed was something different: a tool that can assess and fix already-deployed reports at scale, directly in the cloud, with zero setup friction.

So I built it. On top of the incredible Semantic Link Labs library by Michael Kovalsky, using ipywidgets to render a full interactive UI inside a Fabric Notebook. The result is a tool that runs where your data already lives — in Microsoft Fabric.

The Big Picture — What Can It Do?

The Power BI Fixer v2 is organized into 12 tabs. Each tab is a self-contained workspace for a specific task. Here's the overview:

📊 Report Explorer — See Everything at a Glance

The Report Explorer gives you a hierarchical view of your entire report structure: reports → pages → visuals. Click any visual and you get a live preview of the actual report page, plus all properties (position, size, type, title) — editable in place.

What makes it powerful:

  • Scan mode — Click 🔍 Scan to run all 15 report fixers in assessment mode. Violations appear as ⚠️ badges on each page and visual in the tree. No changes are made — just a full health check.
  • Fix actions — Select an action from the dropdown (Fix Pie Charts, Fix Page Size, Hide Visual Filters, Apply IBCS Theme, ...) and hit Run. The fix is applied, the report is saved back to Fabric, and the preview refreshes.
  • Visual → Model navigation — Click any visual to see which measures and columns it uses. Then jump straight to the Model Explorer to edit that measure's DAX.
  • PBIR auto-detection — If a report is in the old PBIRLegacy format, the tool warns you and offers one-click conversion to the new PBIR format before applying fixes.

📄 Model Explorer — Your Semantic Model Workbench

The Model Explorer is where you work with the semantic model. It gives you a full object tree — tables, columns, measures, hierarchies, calculation groups, and relationships — with editable properties, DAX preview, and table data preview.

The actions dropdown is where it gets interesting. Beyond the standard operations (create measures, create calculated columns, delete objects), there are automated actions that can save hours of manual work:

  • Auto-Create Measures from Columns — Scans all columns with SummarizeBy set and creates proper explicit measures for them. Because implicit measures are the silent killer of good data models.
  • Add PY Measures — Automatically generates Previous Year, ΔPY, and ΔPY% measures for all (or selected) measures. Time intelligence in one click.
  • Add Calendar Table — Creates a calculated calendar table with hierarchies, display folders, and it even proposes the correct relationship to your fact table automatically.
  • Add Time Intelligence & Units Calculation Groups — Common calculation group patterns, pre-built and ready to go.
  • Format All DAX — Formats every DAX expression in the model via the daxformatter.com API. Clean code, zero effort.
  • Prep for AI — Automatically prepares your semantic model for Copilot and AI features — adding descriptions, synonyms, and linguistic metadata.
  • Undo / Redo — Made a mistake? Just hit Undo. Full change history tracking.

📋 Model BPA — 19 Rules, 15 With No Equivalent Elsewhere

The Model Best Practice Analyzer scans your semantic model against 19 rules across 6 categories: Data Types, MDX Settings, Documentation, Formatting, Schema, and Naming. Each finding can be auto-fixed with a single click.

What's special: 15 of the 19 auto-fixers have no standard equivalent in any other tool. Standard BPA implementations can detect these issues — but they can't fix them automatically. The Power BI Fixer can. Rules like "Fix Percentage Format", "Fix Flag Column Format", "Set SortByColumn on Month Columns", "Capitalize Object Names" — these are tedious manual corrections that the Fixer handles in bulk.

You can fix everything at once ("Fix All"), fix an entire rule category ("Fix Rule"), or fix individual findings row by row ("Fix Row"). Your choice.

🌐 Translations — One-Click Multi-Language

The Translations Editor lets you manage metadata translations for your semantic model. Add languages, see all objects in a grid, and — here's the killer feature — hit Auto-Translate to fill in all translations automatically using Azure AI Translator via SynapseML.

No API key setup needed. No cost. SynapseML runs directly in the Fabric Spark environment. Add German translations for 200 objects? 30 seconds.

💾 Memory Analyzer — Know Your Model Size

The Memory Analyzer gives you detailed Vertipaq statistics for your semantic model — broken down by tables, columns, partitions, relationships, and hierarchies. You can see exactly which columns consume the most memory, check cardinality, encoding types, and even Direct Lake residency status.

Color-coded percentage bars make it immediately obvious where the memory hogs are. The "% Table" and "% DB" columns tell you the relative weight of each column — essential for model optimization.

👁 Perspectives — Visual Perspective Management

The Perspective Editor gives you a clean UI for managing model perspectives. Expand tables, check or uncheck individual columns, measures, and hierarchies. Tri-state indicators show you at a glance which tables are fully included, partially included, or excluded.

Create new perspectives, modify existing ones, or delete them — all saved back to the model via XMLA.

🗺 Model Diagram — See Your Relationships

The Model Diagram tab generates an auto-layout SVG diagram of your data model — showing all tables as boxes with their columns and measures, connected by relationship lines. It's read-only, but incredibly useful for quick model comprehension.

✏️ Prototype — Screenshot Your Reports to Excalidraw

The Prototype tab captures every page of your report as a screenshot and assembles them into a visual prototype. You can export to Excalidraw (for whiteboarding and annotation) or SVG (for documentation). It even detects page navigation links and drillthrough relationships between pages.

This is a game-changer for report reviews and prototyping workshops. Take a snapshot of the current state, annotate it in Excalidraw, discuss improvements with stakeholders — all from a single button click.

📐 Delta Analyzer — Inspect Your Lakehouse Tables

For Direct Lake models, understanding your delta table structure matters. The Delta Analyzer tab lets you inspect parquet files, row groups, column chunks, and column statistics for any delta table in your lakehouse.

📄 Report BPA — Catch Report-Level Issues

While the Model BPA focuses on the semantic model, the Report BPA analyzes the report layer itself. It catches issues like unused custom visuals, "Show Items with No Data" enabled on visuals, and report-level measures that should live in the model.

Reports need to be in PBIR format for analysis — but the tool auto-converts PBIRLegacy reports when needed.

⚡ IBCS — International Business Communication Standards

I am passionate about IBCS — the international standards for clear and consistent business reporting. The Power BI Fixer includes dedicated IBCS fixers:

  • Fix IBCS Variance — Transforms column charts into proper IBCS variance charts with relative/absolute variance bars, green/red coloring, and correct axis formatting.
  • Apply IBCS Theme — Applies a clean IBCS-compliant theme to your report.
  • Fix All Charts — Standardizes all chart types to IBCS-friendly formatting: no unnecessary gridlines, proper data labels, clean axes.

Getting Started — 3 Steps

Step 1: Open a Microsoft Fabric Notebook. Run:

%pip install git+https://github.com/KornAlexander/semantic-link-labs.git@feature/pbi-fixer-ui

Step 2: In the next cell, run:

from sempy_labs import pbi_fixer
pbi_fixer()

Step 3: Type your workspace and report name (or leave blank to load everything), then start exploring and fixing.

That's it. No config files. No environment variables. No API keys. Just works.

Requirements

  • Microsoft Fabric workspace (F or P capacity)
  • XMLA read/write endpoint enabled for semantic model operations
  • PBIR format for report-layer fixes (the tool can auto-convert PBIRLegacy reports)
  • Tenant settings: "Export reports as image files" (for Prototype screenshots), "Download reports" (for .pbix export)
